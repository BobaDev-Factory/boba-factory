# Quality Gates (DoR + DoD)

## Definition of Ready (DoR)
- Business need is clear.
- Acceptance criteria are explicit.
- Technical impacts are identified.
- Dependencies are known.
- Scope boundaries are clear (in/out).

## Definition of Done (DoD)
- Implementation completed and reviewed.
- Required tests pass locally and in CI.
- Browser validation is done for UI-impacting scope.
- Documentation updated when behavior/process changed.
- Jira status reflects real delivery state.
- Ticket can move to `Terminé(e)` only with explicit owner approval.
